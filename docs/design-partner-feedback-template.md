# Design-partner feedback

Please answer after completing the external developer test. Short, direct answers are preferred.

1. How many minutes did the no-spend check take from opening the instructions?
2. How many minutes did the paid check take after receiving gateway access?
3. Which step, term, or error message was unclear?
4. Did keeping the payer key behind ACM while binding resource, budget, purpose, receipt, and
   revocation solve a problem you would otherwise need to build?
5. Would you integrate or pay for this control layer? If not, what is the single biggest reason?

Optional:

- runtime and operating system;
- agent framework or workload type;
- one other x402 resource you would want governed next.

Do not include keys, credentials, `.env` files, paid response bodies, or shell history.
